# AI 蔡老师

本地项目，用来承接蔡老师相关论文的题录、摘要、DOI 和可合法访问的全文材料，并生成一个面向研究脉络梳理的 AI 工作台。

## Public GitHub Version

This repository is safe to publish when the ignore rules are respected because it excludes:

- ModelScope / DeepSeek / DashScope API keys and `.env*`
- downloaded PDF files
- raw Web of Science exports
- full-text extracted paper chunks
- local Worker runtime cache and anonymous conversation/usage data

The GitHub Pages frontend in `docs/` calls the protected Cloudflare Worker backend in `worker/`. The Worker keeps the ModelScope token private as a Cloudflare secret, gives each browser an anonymous visitor identity, and enforces hard request limits before calling ModelScope.

The public version uses the compact distillation and evidence summaries generated by:

```bash
npm run build:worker-knowledge
```

It does not publish PDF full text. The local version remains the full workspace for PDF ingestion, full-text indexing, and corpus maintenance.

## Anonymous Guest Isolation

AI 蔡老师 follows the same anonymous-guest pattern as the Thomas distillation project. No login is required. Each browser profile receives its own anonymous visitor id, stored locally in the browser and mirrored through the protected Worker request header/cookie. One visitor's local chat history does not appear in another visitor's browser.

Cloudflare Worker usage counters are keyed by anonymous visitor id and by global day:

- 8 requests per hour per anonymous visitor
- 20 requests per day per anonymous visitor
- 300 requests per day globally
- 2,000 ModelScope free daily calls as a hard upper guard

The Worker uses `MODELSCOPE_API_KEY` as a Cloudflare secret. Do not put the token in `docs/config.js`, frontend code, GitHub Actions logs, README examples, or issue screenshots.

## GitHub Pages + Cloudflare Worker

`docs/config.js` points the static app to:

```js
window.AI_PROF_CHAI_API_BASE = "https://ai-prof-chai-modelscope-api.xinyanzjo.workers.dev";
```

Build the public frontend into `docs/`:

```bash
npm run build:pages
```

Deploy the Worker after logging in to Cloudflare and setting the secret:

```bash
npx wrangler secret put MODELSCOPE_API_KEY
npm run deploy:worker
```

## 当前边界

- Web of Science 登录、验证码、机构权限需要在 Chrome 中由用户完成。
- 项目不绕过付费墙，不保存账号密码，不批量抓取受限 PDF。
- WoS 导出文件放到 `data/wos/` 后运行导入脚本。
- PDF 可放到 `data/pdfs/`，但默认不提交到 git。

## Web of Science 导出建议

在 Web of Science 中通过目标作者身份导出记录时，优先选择：

- Export type: Tab-delimited 或 Plain text / Full Record
- Record content: Full Record and Cited References
- File 放入：`data/wos/`

公开仓库不放具体身份检索清单；需要补数据时请在本地工作区维护。

然后运行：

```bash
npm install
npm run import:wos
npm run dev
```

导入后会生成：

- `data/processed/chai-publications.json`：AI 蔡老师使用的本地语料索引
- `data/processed/target-publications.md`：第一作者/通讯作者论文下载清单
- `data/processed/target-publications.csv`：可核对的下载队列

若已经合法下载到 PDF，请按清单中的文件名放入 `data/pdfs/`，再运行：

```bash
npm run import:wos
npm run index:pdfs
npm run distill
```

项目会自动把对应记录标记为 `pdf-saved`，并把已保存 PDF 变成本地全文索引。无法通过机构权限或开放获取合法访问的全文，只保留题录、摘要、DOI 与下载缺口，不绕过付费墙。

如果最后几篇只能从正常登录浏览器或机构网络手动下载，也可以先把原始 PDF 放进：

```text
data/pdf-inbox/
```

如果 PDF 已经下载到系统的 `Downloads` 文件夹，也可以不用移动。下载后统一运行：

```bash
npm run refresh:pdfs
```

导入器会同时扫描 `Downloads` 和 `data/pdf-inbox/`，校验 PDF 文件头，尝试按 DOI、标题词和年份匹配目标论文，复制到规范的 `data/pdfs/` 文件名，并刷新覆盖率、全文索引、PDF 匹配审计、蒸馏地图、下载包和文献传递包。它只复制能匹配目标论文的 PDF，不会移动或删除原始下载文件。

如果本地页面已经打开，也可以在“PDF 补齐”区直接点“上传PDF”选择已下载的文件，或点“扫描下载”读取 `Downloads` / `data/pdf-inbox/`。这两个入口都会调用同一套刷新流程，完成后自动更新页面上的 PDF 覆盖率、缺失队列和全文索引状态。

如果你准备连续手动补 PDF，可以先开一个自动接收窗口：

```bash
npm run watch:pdfs
```

然后在普通浏览器里打开下载包或快捷入口逐个保存 PDF。监控器会看 `Downloads` 和 `data/pdf-inbox/`，发现新的稳定 PDF 后自动运行 `npm run refresh:pdfs`；对 `Downloads` 里的非目标文件不会在输出里暴露文件名。第一次运行会把已有 PDF 记为“已看过”，之后只处理新下载的文件。如需重置基线，可运行：

```bash
npm run watch:pdfs -- --reset
```

`npm run index:pdfs` 会生成本地全文证据层：

- `data/processed/chai-fulltext-index.json`：本地全文索引，默认被 `.gitignore` 忽略
- `outputs/ai-prof-chai-fulltext-index.md`：可读的索引摘要
- `npm run audit:pdfs` 会生成 `outputs/saved-pdf-audit.md`，用于核对已保存 PDF 是否匹配目标题录

如需继续补齐剩余 PDF，可运行：

```bash
npm run export:missing-pdfs
npm run export:download-pack
npm run export:browser-shortcuts
npm run export:library-request
npm run export:acquisition-pack
npm run export:coverage
npm run export:evidence-pack
npm run eval:assistant
npm run watch:pdfs -- --once
npm run report:status
npm run audit:goal
npm run check:missing-oa
npm run check:missing-metadata
npm run check:cuhk-pure
npm run download:missing-direct
```

这会生成：

- `data/processed/missing-pdf-queue.csv`：剩余 PDF 的核对表
- `outputs/missing-pdf-download-queue.md`：带 DOI、优先级、下一步动作、手动入口和预期文件名的下载队列
- `outputs/missing-pdf-download-pack.html`：可在浏览器中打开的一页式下载包，按优先级排序，可复制文件名和导入命令
- `outputs/missing-pdf-sprint-checklist.md` / `.html`：按成功率排序的 PDF 下载冲刺清单，优先打开最可能用普通浏览器拿到的路线
- `outputs/missing-pdf-browser-shortcuts/`：每条剩余下载路线的 macOS `.webloc` 快捷入口和索引页，方便用普通浏览器逐个打开
- `outputs/missing-pdf-library-request.csv` / `.ris` / `.md` / `.html`：可用于图书馆或文献传递请求的剩余全文清单；HTML 版可逐篇复制申请文本
- `outputs/missing-pdf-acquisition-pack.md` / `.html`：可复制的图书馆/作者请求模板、15 篇逐条证据和行动顺序
- `outputs/missing-pdf-open-access-recheck.md`：对剩余 DOI 的 Unpaywall 开放获取复查和真实 PDF 字节检查
- `outputs/missing-pdf-public-web-recheck.md`：公开网页检索补充记录，说明哪些 PDF 线索被安全验证挡住、哪些相似 PDF 已排除
- `outputs/missing-pdf-scholarly-metadata-recheck.md`：Semantic Scholar / OpenAlex / Crossref 结构化 PDF 元数据复查和字节验证
- `outputs/missing-pdf-cuhk-pure-check.md`：CUHK Pure 页面专项复查，抽取 `citation_pdf_url` 与 `files/*.pdf` 候选并做真实 PDF 字节验证
- `outputs/target-coverage-matrix.csv` / `.md`：51 篇目标论文的覆盖矩阵，整合角色、PDF 状态、全文索引、审计可信度和下一步动作
- `outputs/ai-prof-chai-evidence-pack.md` / `.csv`：36 篇已索引 PDF 的主题证据索引、审计可信度和缺口清单
- `outputs/ai-prof-chai-local-eval.md` / `.json`：不调用外部模型的 AI 蔡老师本地问答评测，检查现有语料能回答什么、哪里需要谨慎
- `outputs/ai-prof-chai-project-status.md`：当前项目状态总控报告
- `outputs/goal-completion-audit.md`：按原始目标逐项核对完成度、证据文件和剩余缺口
- `data/wos-downloads/direct-route-download-report-日期.json`：公开 PDF 直链尝试报告，只保存真实 PDF，权限页或 HTML 页会跳过

本地接口 `/api/missing-pdfs` 会读取这份 CSV，页面中的“PDF 补齐”区会随队列更新；`/api/pdfs/upload` 会接收本地选择的 PDF 并刷新项目；`/api/pdfs/refresh` 会扫描 `Downloads` 和 `data/pdf-inbox/` 并刷新项目；`/api/goal-audit` 会打开目标完成度审计；`/api/assistant/eval` 会打开本地问答评测；`/api/missing-pdfs/download-pack` 会打开 HTML 下载包，`/api/missing-pdfs/sprint-checklist` 会打开冲刺清单，`/api/missing-pdfs/browser-shortcuts` 会打开浏览器快捷入口索引页，`/api/missing-pdfs/acquisition-pack` 会打开合法获取包，`/api/missing-pdfs/open-access-recheck` 会打开 OA 复查报告，`/api/missing-pdfs/public-web-recheck` 会打开公开网页复核报告，`/api/missing-pdfs/library-request/html` 会打开可复制申请文本的一页式文献传递页面，`/api/missing-pdfs/library-request/csv` 与 `/api/missing-pdfs/library-request/ris` 可下载文献传递清单。

`npm run distill` 会生成：

- `data/processed/chai-distillation.json`：AI 蔡老师的机器可读主题/时间线地图
- `outputs/ai-prof-chai-distillation.md`：可直接阅读的研究脉络蒸馏报告

## OpenAlex 公开预索引

Web of Science 登录前，可先生成公开来源候选清单，供后续和 WoS 导出交叉核对：

```bash
npm run fetch:openalex
```

这会生成：

- `data/processed/openalex-chai-publications.json`
- `data/processed/openalex-target-publications.md`
- `data/processed/openalex-target-publications.csv`

若候选记录提供直接 PDF 的开放获取链接，可下载到 `data/pdfs/`：

## AI 配置

本项目优先支持魔搭 ModelScope 的免费 token。`.env.local` 是本机私密配置，已被 `.gitignore` 忽略，不会进入前端页面或仓库。

推荐直接在本地页面的 “AI 蔡老师” 对话区粘贴 token：输入框是密码框，点击 “保存 token” 后会写入 `.env.local`，再点击 “测试连接” 确认魔搭模型是否真的可用。这个流程不会在聊天里显示 token，也不会把 token 返回到页面结果里。

也可以用本地隐藏输入助手配置，不要把 token 发在聊天里：

```bash
npm run setup:ai
npm run check:ai
```

也可以只检查当前配置状态：

```bash
npm run setup:ai -- --check
```

在 `.env.local` 中填写：

```text
AI_PROF_CHAI_AI_PROVIDER=modelscope
AI_PROF_CHAI_MODELSCOPE_API_TOKEN=你的魔搭 token
AI_PROF_CHAI_MODELSCOPE_MODEL=Qwen/Qwen3-235B-A22B-Instruct-2507
AI_PROF_CHAI_MODELSCOPE_API_BASE=https://api-inference.modelscope.cn/v1
```

如果你已经在系统环境里保存过免费 token，也可以直接用 `MODELSCOPE_API_TOKEN` 或 `MODELSCOPE_API_KEY`，项目会自动识别。根据魔搭 API-Inference 官方说明，它提供 OpenAI API 兼容接口，当前默认地址使用 `https://api-inference.modelscope.cn/v1`。

填完 token 后，可以在页面点击 “测试连接”，或运行 `npm run check:ai` 检查 AI 对话是否已接通。这个命令不会打印 token，只会显示模型是否成功返回。`npm run setup:ai` 也不会打印完整 token，只显示掩码确认。

填完后重启本地服务，页面右上角会显示当前模型；未配置 token 时，页面会用本地语料规则给出简短回答，方便先检查语料导入是否正确。
